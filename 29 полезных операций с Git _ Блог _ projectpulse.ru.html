29 полезных операций с Git

03.12.2021

1. Настройка имени пользователя и электронной почты

Имя пользователя треубется для связи коммитов с вашим именем. И да, это совсем не то же самое что имя аккаунта на Github. Установить имя пользователя можно следующей командой:

git config --global user.name "developer"

Аналогичным способом можно указать адрес электронной почты, который будет сопровождать все ваши будущие коммиты:

git config --global user.email "dev@projectpulse.ru"

2. Кеширование данных для авторизации

При использовании авторизации по логину и паролю в работе с удалённым репозиторием можно закешировать авторизационные данные, чтобы не вводить их каждый раз снова:

git config --global credential.helper cache

3. Инициализация репозитория

Создать новый репозиторий или переинициализировать существующий можно с помощью параметра init. Такая операция создаёт специальную скрытую директорию содержащую всё необходимое для существования репозитория:

git init

4. Добавление файлов к коммиту: всех или по одному

Для добавления одного файла к коммиту:

git add somefile.js

Если вместо имени файла указать просто точку ., то рекурсивно будут добавлены все файлы в текущей и вложенных директориях:

git add .

5. Статус репозитория

Посмотреть текущее состояние репозитория можно с помощью параметра status. Команда выводит список добавленных и не добавленных к коммиту файлов, а также ещё неотслеживаемые файлы.

git status

6. Создание коммита

Создать коммит с однострочным сообщением можно с помощью аргумента -m команды commit:

git commit -m "Repository initialization"

Если же просто вызвать команду git commit без аргументов, то будет предложено открыть текстовый редактор для ввода сообщения коммита.

7. Просмотр истории коммитов с изменениями

Параметр log выведет список последних коммитов репозитория. Добавив аргумент -p можно также смотреть изменения сделанные каждым коммитом:

git log -p

8. Просмотр одного коммита

Детальную информацию по отдельному коммиту можно посмотреть параметром show. Аргументом для команды должен быть передан хеш коммита, но копировать весь хеш целиком не обязательно: можно обойтись первыми шестью символами:

git show fa21ee9

9. Просмотр изменений перед созданием коммита

Посмотреть все изменения в репозитории которые ещё не были добавлены к коммиту можно с помощью команды

git diff

Если требуется просмотреть изменения которые были добавлены к коммиту перед его созданием:

git diff --staged

Также командой можно смотреть изменения отдельного файла:

git diff index.php

10. Удаление файла из репозитория

Удалить файл из репозитория и отключить его дальнейшее отслеживание можно параметром rm:

git rm code/index.php

11. Переименовывание и перемещение файлов

Переименовать файл или директорию можно параметром mv:

git mv etc/view.php newdir

12. Откат изменений файлов

Для отмены изменений которые ещё не были добавлены к коммиту:

git checkout sometest.txt

Если вызвать команду без указания конкретного файла или директории, то будут отменены все изменения в файлах к состоянию последнего коммита.

git checkout

Если файл уже был добавлен к коммиту, то для отмены понадобится параметр reset:

git reset HEAD sometest.txt

Так же команду можно использовать без указания файлов, для отмены все добавленных к коммиту файлов.

13. Исправление последнего коммита

Аргумент --amend параметра commit позволяет исправлять ошибки последнего коммита, путём автоматического создания нового исправленного коммита в истории. Пример исправления сообщения последнего коммита:

git commit --amend -m "Новое сообщение для последнего коммита"

Кроме изменения сообщения можно вносить изменения и в сами файлы, например если мы забыли добавить директорию к последнему коммиту:

git add forgotten_dir
git commit --amend --no-edit

Флаг --no-edit используется когда не требуется вносить изменения в сообщение коммита.

Осторожно! Не исправляйте коммиты удалённых репозиториев
Инструмент отлично подходит для исправления коммитов, которые пока остаются в локальном репозитории. Но исправление коммитов которые уже были отправлены в удалённый общедоступный репозиторий может создать определённые проблемы без понимания логики работы.

14. Откат изменений коммита

Откатить изменения последнего коммита можно параметром revert. Так будет создан новый коммит, полностью обратный предыдущему, что позволит отменить изменения внесённые последний коммитом:

git revert HEAD

Таким образом можно создать обратный коммит не только последнему, но и любому из истории, указав хеш нужного коммита:

git revert fa21ee9

15. Создание и переключение к новой ветке

Создать новую ветку легко:

git branch new-feature

После создания новой ветки, переключиться на неё можно командой:

git checkout -b new-feature

16. Список веток репозитория

Посмотреть список веток можно командой:

git branch

Также можно посмотреть все ветки удалённого репозитория:

git branch -a

17. Удаление ветки

После того как работа с веткой завершена и она слита с другой основной веткой, её можно удалить:

git branch -d new-feature

При попытке удалить ветку, которая ещё не была слита с другой, будет возвращена ошибка. В этом случае для принудительного удаления:

git branch -D new-feature

Этими командами мы удаляем ветки только в локальном репозитории. Если же требуется удалить ветку в удалённом репозитории:

git push origin --delete new-feature

18. Слияние двух веток

Для операций слияния используется параметр merge:

git merge new-feature

Данная команда запустит процесс слияния другой указанной ветки в текущую.

Если требуется сделать слияние путём создания нового коммита, потребуется указать флаг --no-ff:

git merge --no-ff new-feature

19. Просмотри истории коммитов в виде графа

Параметр log позволяет вывести историю коммитов в виде графа:

git log --graph --oneline --decorate

Где флаг --graph используется для рисования истории ASCII-символами, а дополнительные флаги --oneline и --decorate просто для удобства просмотра истории.

Для просмотра истории коммитов не только текущей ветки, а всех, есть ещё один дополнительный флаг --all:

git log --all --graph --oneline --decorate

20. Отмена конфликтного слияния

При возникновении конфликта в процессе слияния можно воспользоваться флагом --abort вместе с вызовом параметра merge:

git merge --abort

Данная команда завершит остановленный процесс слияния путём отмены всех изменений и возвращением к состоянию перед запуском команды git merge.

Также можно воспользоваться командой git reset которая просто отменит все изменения файлов к последнему стабильному состоянию.

21. Добавление удалённого репозитория

Для добавления удалённого репозитория используется параметр remote add с указанием имени удалённого репозитория и его URL:

git remote add origin https://github.com/somerep

22. Просмотр адресов удалённых репозиториев

Посмотреть все удалённые репозитории можно командой 

git remove -v

Сам список репозитеориев с адресами содержится в файле .git/config.

23. Получение дополнительной информации об удалённом репозитории

Получить информацию об удалённом репозитории можно параметром remote show и именем репозитория, например origin.

git remote show origin

24. Отправка изменений на удалённый репозиторий

Для отправки изменений на удалённый репозиторий используется параметр push с именем репозитория и ветки:

git push origin master

25. Загрузка изменений с удалённого репозитрия

Параметр pull загружает изменения с удалённого репозитория и сразу делает слияние изменений с локальным репозиторием:

git pull

Для получения дополнительной ифнормации о процессе пригодится флаг --verbose:

git pull --verbose

26. Слияние удалённого репозитория с локальным

Кроме веток, есть возможность слияния репозитория целиком:

git merge origin

27. Отправка новой ветки на удалённый репозиторий

Для отправки свежесозданной ветки на удалённый репозиторий, необходимо вызвать параметр push с флагом -u:

git push -u origin new-feature

29. Удаление ветки на удалённом репозитории

Для удаления же ветки понадобится флаг --delete:

git push --delete origin new-feature
